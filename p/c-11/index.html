2.3 C++11中的std::initializer_list

文档

initializer_list 和 列表初始化的区别

  • initializer_list 是天然的类型识别，{}中可以是任意多个对象

    如果把 {} 给auto或没有具体类型的模版时，会被识别成 initializer_list

    1

    auto il = { 10, 20, 30, 40 };

  • 列表初始化中的{} 本质是构造，{}中的值要跟类型的构造参数匹配

initializer_list 的本质

initializer_list类本质是在底层开一个数组空间，将数据拷贝过来，让initializer_list内部的两个指针分别指向数组的开始和结束，所以 initializer_list 的 {} 中可以是任意多个值

initializer_list 相关接口

  • size：就是用两个指针相减得到中间的数据个数
  • begin 和 end ：返回的迭代器就是原生指针，begin返回指向开始位置的指针，end返回指向结束位置的指针

支持迭代器就支持范围 for

initializer_list 的主要用途

使用列表初始化对象容器存在的问题：列表初始化要求{}中的值要和构造参数匹配，如果我们想用任意多个值去构造初始化，那么我们需要实现很多个构造函数才能支持，很不方便。

而C++11提出initializer_list就能很好解决这个问题。只要让容器支持一个std:initializer_list的构造函数，也就支持任意多个值进行初始化了

STL中的容器都增加了一个 std::initializer_list 的构造，因此支持任意多个值进行初始化

3.右值引用和移动语义

引用就是给对象取别名

C++98中就有引用的语法，C++11又新增了右值引用的语法，所以我们一般把C++11之前的引用叫左值引用

3.1 左值和右值

左值

  • 左值是一个数据表达式（如变量名或解引用的指针），一般是有持久状态，存储在内存中
  • 左值既可以出现在赋值符号的左边，也可以出现在右边，即左值可以修改也可以不能被修改
  • 左值可以取地址
  • 变量表达式都是左值

右值

  • 右值也是一个表示数据的表达式（如字面值常量、表达式求值过程中创建的临时对象、匿名对象等）

  • 右值只能出现在赋值符号的右边，即右值不能被修改

  • 右值不能取地址

注意：判断是左值还是右值和是否可以修改无关，核心在于是否可以取地址

比如const修饰符后的是左值，因为可以取地址

3.5 右值引用和移动语义

3.5.1左值引用存在的问题

回顾左值引用的使用场景：

  1. 做参数减少拷贝，改变引用对象的同时改变被引用对象
  2. 做返回值减少拷贝

但是不是任何场景下都能用左值引用，比如函数内定义的局部对象就不能用引用返回。

所以我们以前遇到传值返回开销很大的情况，只能被迫用输出型参数解决，即先在函数外定义好所需变量，然后让函数参数用左值引用来接收该变量。

那用右值引用可以解决左值引用做返回值的问题吗？也不能。因为函数执行结束后，函数内部定义的局部对象就销毁了，右值引用同样无法引用一个已经销毁的对象。但是我们接下来要学的移动构造可以解决这个问题

namespace dyy
{
	class string
	{
	public:
		typedef char* iterator;
		typedef const char* const_iterator;

		iterator begin()
		{
			return _str;
		}

		iterator end()
		{
			return _str + _size;
		}

		const_iterator begin() const 
		{
			return _str;
		}

		const_iterator end() const 
		{
			return _str + _size;
		}

		string(const char* str = "")
			:_size(strlen(str))
			, _capacity(_size)
		{
			cout << "string(const char* str)构造" << endl;
			_str = new char[_capacity + 1];
			strcpy(_str, str);
		}

		void swap(string& s)
		{
			::swap(_size, s._size);
			::swap(_str, s._str);
			::swap(_capacity, s._capacity);
		}

		string(const string& s)
			:_str(nullptr)
		{
			cout << "string(string& s)拷贝构造" << endl;
			reserve(s._capacity);
			for (auto ch : s)
			{
				push_back(ch);
			}
		}

		void reserve(size_t n)
		{
			if (n > _capacity)
			{
				char* tmp = new char[n + 1];
				if (_str)
				{
					strcpy(tmp, _str);
					delete[] _str;
				}
				_str = tmp;
				_capacity = n;
			}
		}

		void push_back(char ch)
		{
			if (_size >= _capacity)
			{
				size_t newcapacity = _capacity == 0 ? 4 : _capacity * 2;
				reserve(newcapacity);
			}

			_str[_size] = ch;
			++_size;
			_str[_size] = '\0';
		}

		string& operator+=(char ch)
		{
			push_back(ch);
			return *this;
		}

		const char* c_str() const
		{
			return _str;
		}

		size_t size() const
		{
			return _size;
		}
	private:
		char* _str = nullptr;
		size_t _size = 0;
		size_t _capacity = 0;
	};
}

template<class T>
struct ListNode
{
  ListNode<T>* _next;
  ListNode<T>* _prev;

  T _data;

  ListNode(const T& data = T())
      :_next(nullptr)
          ,_prev(nullptr)
          ,_data(data)
      { }

  //移动构造
  ListNode(const T&& data)
      :_next(nullptr)
          , _prev(nullptr)
          , _data(move(data)) // 注意向下传递时要用move(左值)
      {}
};

template<class T, class Ref, class Ptr>
struct ListIterator
{
  typedef ListNode<T> Node;
  typedef ListIterator<T, Ref, Ptr> Self;
  Node* _node;

  ListIterator(Node* node)
      :_node(node)
      { }

  Self& operator++()
  {
      _node = _node->_next;
      return *this;
  }

  Ref operator*()
  {
      return _node->_data;
  }

  bool operator!=(const Self& it)
  {
      return _node != it._node;
  }
};

template<class T>
class list
{
  typedef ListNode<T> Node;
  public:
  typedef ListIterator<T, T&, T*> iterator;
  typedef ListIterator<T, const T&, const T*> const_iterator;

  iterator begin()
  {
      return iterator(_head->_next);
  }

  iterator end()
  {
      return iterator(_head);
  }

  void empty_init()
  {
      _head = new Node();
      _head->_next = _head;
      _head->_prev = _head;
  }

  list()
  {
      empty_init();
  }

  void push_back(const T& x)
  {
      insert(end(), x);
  }

  //右值引用版本
  void push_back(T&& x)
  {
      insert(end(), move(x)); // 注意向下传递的时候要用move(左值)
  }

  iterator insert(iterator pos, const T& x)
  {
      Node* cur = pos._node;
      Node* newnode = new Node(x);
      Node* prev = cur->_prev;

      // prev newnode cur
      prev->_next = newnode;
      newnode->_prev = prev;
      newnode->_next = cur;
      cur->_prev = newnode;

      return iterator(newnode);
  }

  //右值引用版本
  iterator insert(iterator pos, T&& x)
  {
      Node* cur = pos._node;
      //注意这里要用move(左值)，因为右值引用变量的属性还是左值
      Node* newnode = new Node(move(x));
      Node* prev = cur->_prev;

      // prev newnode cur
      prev->_next = newnode;
      newnode->_prev = prev;
      newnode->_next = cur;
      cur->_prev = newnode;

      return iterator(newnode);
  }
  private:
  Node* _head;
};

3.6 类型分类

C++11以后，进一步对值类别进行了划分：

右值被拆分为纯右值（pure value，简称prvalue）和将亡值（expiring value， 简称xvalue）。

  • 纯右值：天生没有名字的临时对象，比如字面值常量、没有名字的临时对象/计算结果等。它没有实体名字，没办法通过它修改原资源。

  • 将亡值：本来有名字，但是被标记为“马上就要销毁”、专门用来转移资源的对象。比如move(左值)、static_cast<右值引用>(左值)处理后，对象就变成了将亡值 。它有明确的资源归属（本来是某个对象的资源），但是被标记为即将销毁，所以编译器会优先调用移动构造，直接转移它的资源。

泛左值（generalized value，简称 glvalue）：能找到资源位置的值，包含左值和将亡值。

纯右值只是一个临时结果，没办法定位到具体资源的地址

3.8 完美转发

在之前学习万能引用的过程中，我们发现一个问题：函数模版内部的参数t，本身是左值（因为引用变量表达式都是左值，不管是左值引用变量表达式还是右值引用变量表达式，都是左值），如果直接把 t 传给其它函数，就会丢失传过来的实参的原本属性（传过来的实参可能是左值、右值、const左值、const右值），导致不管原来实参是什么类型，最终都会匹配到左值引用版本的函数重载。

这背后的关键是函数模板与普通重载函数的参数接收机制不同：函数模板会通过模板参数T推导，记录实参的原始属性；而普通重载函数不会记录这些信息，只认“传入的表达式本身是左值还是右值”。正因为普通函数只看眼前的表达式属性，直接传左值属性的 t 才会匹配错误重载

完美转发能识别原实参属性的核心，是依托万能引用的推导结果 + 引用折叠规则：

1）万能引用接收实参时，会将原实参的左值 / 右值属性 “编码” 到 T 的推导类型中 —— 传左值时 T 推导为左值引用，传右值时 T 推导为非引用类型，这一步就已经保留了原实参的属性标记。

2）而std::forward底层通过static_cast<T&&>(t)结合引用折叠，根据 T 的推导类型反向还原：若 T 是左值引用，经引用折叠后转为左值引用返回，若 T 是非引用类型，则直接转为右值引用返回，本质是靠 T 的推导结果 “记住” 原实参类型，再通过引用折叠精准还原转发。

4.3 emplace系列接口

基本介绍

C++11 之后，容器里所有带插入功能的接口，都新增了对应的 emplace 系列接口

比如原来有 push_back的，现在多了 emplace_back; 有 push_front 的，多了 emplace_front; 有 insert 的，多了 emplace 接口

template <class... Args> void emplace_back (Args&&... args); template <class... Args> iterator emplace (const_iterator position,Args&&... args);

emplace比原先的插入接口更高效，但是多参数的时候不能使用{}来插入

**注意：**传递参数包过程中，如果是 Args&&… args 的参数包，要用完美转发参数包，方式如下std::forward\<Args>(args)...，否则编译时包扩展后右值引用变量表达式就变成了左值

新增的emplace系列接口和原先的插入接口（insert、push_back 等）的区别：

1）emplace系列的接口均为模板可变参数，而原先的插入接口是普通函数（不是通过参数推演，类模版实例化，value_type就实例化）

2）原先的插入接口要么插入左值，要么插入右值，而emplace系列接口用的是万能引用，可以接收任意类型的参数

3）原先的插入接口只能接收单个value_type 对象，而emplace系列接口可以传一个参数包，这个参数包可以是零到多个参数

emplace 系列接口在某些场景下兼容旧插入接口，同时也在一些场景下比 push_back 更高效

我们以push_back和emplace_back为例：

1）emplace_back 传左值时走拷贝构造、传右值时走移动构造，效果与push_back完全一样

2）但当传入构造 T 的原始参数 时，emplace_back 可以直接构造，而push_back 需要构造+移动构造+析构

C++11支持单参数的构造走隐式类型转换，我们以string为例，如果直接传"11111111"，

  • 因为emplace_back是模版，会自动推导参数是const char* 类型的，然后直接走构造
  • 而push_back就是普通函数，value_type已经确定是string了，所以push_back函数接收到"11111111"参数后，会先拷贝构造一个string对象，然后因为该对象是临时对象，会直接匹配到右值引用版本的push_back，转移资源后，该对象销毁

容器存复合类型（如 pair）时，emplace_back 仍兼容旧接口的左值 / 右值传入方式。

1）emplace_back(kv)（左值）：走拷贝构造，效果与 push_back(kv) 完全一致；

2）emplace_back(move(kv))（右值）：走移动构造，效果与 push_back(move(kv)) 完全一致。

#pragma once

namespace dyy
{
    template<class T>
    struct ListNode
    {
        ListNode<T>* _next;
        ListNode<T>* _prev;

        T _data;

        ListNode(const T& data = T())
            :_next(nullptr)
            , _prev(nullptr)
            , _data(data)
        {}

        ListNode(T&& data)
            :_next(nullptr)
            , _prev(nullptr)
            , _data(move(data))
        {}

        // 不确定传过来的是左值还是右值，需要用完美转发
        template<class... Args>
        ListNode(Args&&... args) 
            :_next(nullptr)
            , _prev(nullptr)
            , _data(std::forward<Args>(args)...) 
        {}
    };

    template<class T, class Ref, class Ptr>
    struct ListIterator
    {
        typedef ListNode<T> Node;
        typedef ListIterator<T, Ref, Ptr> Self;
        Node* _node;

        ListIterator(Node* node)
            :_node(node)
        {
        }

        Self& operator++()
        {
            _node = _node->_next;
            return *this;
        }

        Ref operator*()
        {
            return _node->_data;
        }

        bool operator!=(const Self& it)const 
        {
            return _node != it._node;
        }
    };

    template<class T>
    class list
    {
        typedef ListNode<T> Node;
    public:
        typedef ListIterator<T, T&, T*> iterator;
        typedef ListIterator<T, const T&, const T*> const_iterator;

        iterator begin()
        {
            return iterator(_head->_next);
        }

        iterator end()
        {
            return iterator(_head);
        }

        void empty_init()
        {
            _head = new Node();
            _head->_next = _head;
            _head->_prev = _head;
        }

        list()
        {
            empty_init();
        }

        //左值引用版本
        void push_back(const T& x)
        {
            insert(end(), x);
        }

        //右值引用版本
        void push_back(T&& x)
        {
            insert(end(), move(x));
        }

        template <class... Args>
        void emplace_back(Args&&... args)
        {
            emplace(end(), std::forward<Args>(args)...);
        }

        iterator insert(iterator pos, const T& x)
        {
            Node* cur = pos._node;
            Node* newnode = new Node(x);
            Node* prev = cur->_prev;

            // prev newnode cur
            prev->_next = newnode;
            newnode->_prev = prev;
            newnode->_next = cur;
            cur->_prev = newnode;

            return iterator(newnode);
        }

        iterator insert(iterator pos, T&& x)
        {
            Node* cur = pos._node;
            Node* newnode = new Node(move(x));
            Node* prev = cur->_prev;

            // prev newnode cur
            prev->_next = newnode;
            newnode->_prev = prev;
            newnode->_next = cur;
            cur->_prev = newnode;

            return iterator(newnode);
        }

        template <class... Args>
        iterator emplace(iterator pos, Args&&... args)
        {
            Node* cur = pos._node;
            Node* newnode = new Node(std::forward<Args>(args)...);
            Node* prev = cur->_prev;

            // prev newnode cur
            prev->_next = newnode;
            newnode->_prev = prev;
            newnode->_next = cur;
            cur->_prev = newnode;

            return iterator(newnode);
        }
    private:
        Node* _head;
    };
}

	class string
	{
	public:
		typedef char* iterator;

		iterator begin()
		{
			return _str;
		}

		iterator end()
		{
			return _str + _size;
		}

		const_iterator begin() const 
		{
			return _str;
		}

		const_iterator end() const 
		{
			return _str + _size;
		}

		string(const char* str = "")
			:_size(strlen(str))
			, _capacity(_size)
		{
			cout << "string(const char* str) - 构造" << endl;
			_str = new char[_capacity + 1];
			strcpy(_str, str);
		}

		void swap(string& s)
		{
			::swap(_size, s._size);
			::swap(_str, s._str);
			::swap(_capacity, s._capacity);
		}

		string(const string& s)
			:_str(nullptr)
		{
			cout << "string(string& s) --- 拷贝构造" << endl;
			reserve(s._capacity);
			for (auto ch : s)
			{
				push_back(ch);
			}
		}

		//移动构造
		string(string&& s)
		{
			cout << "string(string&& s) --- 移动构造" << endl;
			swap(s);
		}

		//移动赋值
		string& operator=(string&& s)
		{
			cout << "string& operator=(string&& s) --- 移动赋值" << endl;
			swap(s);
			return *this;
		}

		~string()
		{
			cout << "~string()" << endl;
			delete[] _str;
			_str = nullptr;
		}

		char& operator[](size_t pos)
		{
			assert(pos < _size);
			return _str[pos];
		}

		void reserve(size_t n)
		{
			if (n > _capacity)
			{
				char* tmp = new char[n + 1];
				if (_str)
				{
					strcpy(tmp, _str);
					delete[] _str;
				}
				_str = tmp;
				_capacity = n;
			}
		}

		void push_back(char ch)
		{
			if (_size >= _capacity)
			{
				size_t newcapacity = _capacity == 0 ? 4 : _capacity * 2;
				reserve(newcapacity);
			}

			_str[_size] = ch;
			++_size;
			_str[_size] = '\0';
		}

		string& operator+=(char ch)
		{
			push_back(ch);
			return *this;
		}

		const char* c_str() const
		{
			return _str;
		}

		size_t size() const
		{
			return _size;
		}
	private:
		char* _str = nullptr;
		size_t _size = 0;
		size_t _capacity = 0;
	};

总结

  1. 兼容旧接口：emplace 系列（emplace_back/emplace）传左值→拷贝构造、传右值→移动构造，和 insert/push_back 效果完全一样；
  2. 高效核心：传 “构造对象的参数”（比如"苹果", 1）时，emplace 直接在容器内构造，无临时对象开销；而旧接口要先造临时对象，再移动 + 析构；
  3. 小坑提醒：emplace 不支持{}初始化列表（比如emplace_back({...})），要传明确的构造参数（比如emplace_back("苹果", 1)）；
  4. 复合类型通用：哪怕容器存 pair 这种复合类型，上面的规则全适用。

能传构造参数就用 emplace，想传{}或现成对象就用旧接口 /emplace 都行（兼容），优先选 emplace

7.4 lambda的原理

lambda表达式的底层本质是仿函数对象，我们写了一个lambda后，编译器会生成一个对应的仿函数类：

  • 仿函数的类名是编译器内部按一定规则生成的，保证不同的 lambda 生成的类名不同
  • lambda的参数、返回类型、函数体会直接对应仿函数operator()的参数、返回类型、函数体
  • lambda 捕捉列表里的变量会成为仿函数类的成员变量，捕捉时相当于把变量作为构造函数的实参，初始化这些成员变量（隐式捕捉时，编译器只会传递函数体内用到的变量）

lambda 和 范围 for类似，都不是全新的语法，而是语法糖。lambda底层基于已有的 仿函数机制实现，让代码写起来更简洁，但本质还是编译器帮我们生成了仿函数类

	function<int(int, int)> f2 = Functor();
	function<int(int, int)> f3 = [](int a, int b) {return a + b; };

	cout << f1(1, 1) << endl; // 2
	cout << f2(1, 1) << endl; // 2
	cout << f3(1, 1) << endl; // 2

	return 0;

int main()
{

	//1.包装静态成员函数

	//注意：非静态的成员函数还有一个隐含的this指针
	//写法一：可调用对象的第一个参数写成指针，调用时传对象地址
	function<double(Plus*, double, double)> f5 = &Plus::plusd;
	Plus ps;
	cout << f5(&ps, 1.1, 1.1) << endl; // 22

    function<double(Plus, double, double)> f6 = &Plus::plusd;

    Plus ps2;
    cout << f6(ps2, 1.1, 1.1) << endl; // 22

    //也可以传匿名对象
    cout << f6(Plus(), 1.1, 1.1) << endl; // 22

    //不过，自定义类型的对象最好还是写成右值引用
    function<double(Plus&&, double, double)> f7 = &Plus::plusd;

包装非静态成员的时候需要注意：

1）非静态成员函数有一个隐含的this指针。

2）定义function的时候，可调用对象的第一个参数既可以传指针也可以传对象，因为function包装后，还是需要用指针去调用成员函数，指针可以调用成员函数，对象也可以调用成员函数。

包装器本质是编译器推导类型。

**所以，为什么会有包装器呢？**本质是因为C++里的可调用对象的类型复杂且多：

  • 函数指针的类型写起来比较麻烦，比如把函数f 的地址赋值给指针变量 pf，就得写成这样 int(*pf)(int, int）=f

  • lambda在语法层面上我们没法控制类型，它的类型是编译器自动生成的

  • 类的成员函数，需要对象或指针去调用，所以类型写起来也很麻烦

而包装器能将这些可调用对象的类型统一成 std::function 类型。可调用对象包装后，就都能用同一个类型的变量存储、传递，并且用法也统一了

8.1bind

基本介绍

1）bind是一个函数模版

2）bind也是一个可调用对象的包装器，可以把bind看做一个函数适配器：我们给它传一个可调用对象，它做一些处理，然后它会返回一个新的可调用对象

3）bind主要是用来调整可调用对象的参数列表的，比如调整参数的个数和参数的顺序

4）bind的定义也是在 <functional>这个头文件中

bind的使用

调用bind的一般形式：

1

auto newCallable = bind(callable, arg_list);
  • callable：可调用对象
  • arg_list：一个以逗号分隔的参数列表，对应给定的callable的参数
  • bind的返回值是一个可调用对象（bind虽然是一个函数模版，但是内部其实是生成可调用对象）

当调用 newCallable(实参) 时，该新对象会自动调用原本的 callable，并且将 arg_list 中的参数（固定值直接使用，占位符用 newCallable 接收的实参替换），一并传递给原本的 callable。

arg_list中的参数可以传 _1、_2、_3等，这些参数是占位符，_1永远代表newCallable的第1个实参，_2永远代表newCallable的第2个实参，以此类推

_1 、_2、 _3 这些占位符是封装在placeholders这个命名空间里的

#include <iostream>
#include <functional>

using namespace std;

using placeholders::_1;
using placeholders::_2;
using placeholders::_3;

int main()
{
    //计算复利的lambda
    //复利：前一年的利息会变成第二年的本金
    auto func1 = [](double rate, double money, int year)->double {
        double ret = money;
        for (int i = 0; i < year; i++)
        {
            ret += ret * rate;
        }
        return ret - money;
    };

    //绑死一些参数，实现支持不同年华利率，不同金额和不同年份计算出复利的结算利息
    function<double(double)> func3_1_5 = bind(func1, 0.015, _1, 3);
    function<double(double)> func5_2_5 = bind(func1, 0.025, _1, 5);
    function<double(double)> func10_2_5 = bind(func1, 0.025, _1, 10);
    function<double(double)> func20_3_5 = bind(func1, 0.035, _1, 30);
    cout << func3_1_5(1000000) << endl;//45678.4
    cout << func5_2_5(1000000) << endl;//131408
    cout << func10_2_5(1000000) << endl;//280085
    cout << func20_3_5(1000000) << endl;//1.80679e+06
        
	return 0;
}
