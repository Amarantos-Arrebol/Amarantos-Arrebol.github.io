3.2cin-cout性能问题

通常scanf和prinf比cin和cout快

**原因：**cin和cout由于要考虑兼容C语言的输入和输出，封装实现的更加复杂

这种差异在大多数应用场景中可以忽略不计，但是在竞赛中，当输入输出数据量

较大的时候，使用cin和cout就可能出现Time Limit Exceeded

总结性能低的原因：

  1. C++中为了支持混合使用cin/cout和printf/scanf，C++标准库默认会将cin/cout

    等C++流对象与stdin/stdout等C标准库的流对象同步在一起。这种同步操作意味着每次使用cin/cout时，都会自动刷新C标准库的缓冲区，以确保C++和C的I/O是一致的

  2. 在默认情况下，cin和cout之间存在一种绑定关系。这种绑定关系意味着每当cin

    读取数据时，任何之前通过cout输出的内容都会被强制刷新到屏幕上。这种绑定

    也可能导致性能问题，特别是在需要频繁读取大量数据的情况下

![73673906487](【从零开始学C++】03输入输出（下）+ 函数.assets/1736739064879.png)

1.函数是什么

函数就是一个完成某项特定的任务的一小段代码

函数的好处：

  1. 模块化开发

    一个大的功能，总能拆分成各种子功能，每个子功能都可以设计成一个函数

    每个函数可以作为一个独立的模块存在，程序的逻辑更加清晰、明确

  2. 代码可以复用

    只需要定义出一个函数，需要这个功能的地方，直接调用函数就行

  3. 方便多个程序员之间协作开发，方便程序的多个模块之间互相交互

  4. 代码的编写、阅读、调试、维护都更加容易

2.函数的分类

2.1库函数

库函数是标准库中提供的现成的函数，我们只要学习函数的功能，就能直接使⽤。

有了库函数，⼀些常⻅的功能就不需要程序员⾃⼰实现了，⼀定程度提升了效率；同时库

函数的质量和执⾏效率上都更有保证。

编译器的标准库中提供了⼀系列的库函数，这些库函数根据功能的划分，都在不同的头⽂

件中进⾏了声明。

C/C++官⽅参考⼿册：https://zh.cppreference.com/w/cpp/header

C/C++第三⽅⽹站：https://legacy.cplusplus.com/reference/

#include <string>

bool is_prime(int m)
{
	//素数：只能被1和自身整数的数
	if(m <= 1)
		return false;
		
	//产生2~sqrt(m)之间的数，去试除m 
	for(int i = 2; i <= sqrt(m); i++)
	{
		if(m % i == 0)
			return false;	
	} 
	return true;
}

int reverse_int(int k)
{
	if(k <= 9)
		return k;
	int ret = 0;
	while(k)
	{
		ret = ret * 10 + k % 10;
		k /= 10;
	}
	return ret;
}

int main()
{
//	思路：
//	1.输入M、N
//	2.产生[M，N]之间的数 for(int i = m; i <= n; i++)
//	3.判断i是否是真素数 
//		1)i是素数
//		2)i的反序也是素数 
//			1)产生反转后的数
//			2)转成string类型，reverse()，stoi 
	int M, N;
	int flag = 0;
	cin >> M >> N;
	
	for(int i = M; i <= N; i++)
	{
		//产生i的反序数字 i2 
		int i2 = reverse_int(i);
		//判断i和i2是否同时为素数 
		if(is_prime(i) && is_prime(i2))
		{
			if(flag)
				cout << ",";
			cout << i;
			flag = 1;
		}
	}
	if(flag == 0)
		cout << "No" << endl;
	
	return 0;
}

7.2 递归思想

递归可以把一个大型复杂问题层层转化为一个与原问题相似，但规模较小的子问题

来求解，直到子问题不能再被拆分，可以直接求解，递归就结束了。

递归的思考方式就是把大事化小的过程

递归就是递推回归的意思

7.3递归的必要条件

递归在书写的时候，需要满足两个必要条件

  1. 递归存在限制条件，当满足这个限制条件的时候，递归就不再继续
  2. 每次递归调用后会越来越接近这个限制条件
