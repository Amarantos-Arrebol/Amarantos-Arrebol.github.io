1.2 操作系统的历史

  • 诞生的背景

    随着计算机越来越复杂，在上面并行跑的任务越来越多

    所以必须让操作系统可以自动化地处理各种各样的任务

    • 同时计算机不仅仅可以做计算任务，还可以做数据保存等更复杂的任务

      这个时候就必须要求有新的一套系统，能把文件进行保存

    • 计算机在发展的过程中，各种任务都在跑，都要消耗内存

      所以必须把内存管理起来

    • 后来也必须外接各种硬件设备，比如话筒，键盘，显示器，鼠标，摄像头，外设等

  • 计算机+操作系统 进入校园

1. ls指令

ls 显示当前路径下，文件名称。pwd 查看当前路径

当前路径下，默认是空的，我们可以使用touch创建一个文件

还可以使用 mkdir 创建一个目录/文件

ls 可以理解成list的意思，默认显示当前路径下的所有普通文件和子目录

  • 普通文件

  • 目录文件

    ls 查看文件的文件名，文件名不是文件的内容，是文件的属性

ls -l 显示当前目录下，更详细的文件属性信息

ls -a 显示隐藏文件

Linux中的选项是可以组合传入的，也可以分开传入

ls -l -a 效果和 ls -la 还有 ls -al 相同

a可以理解成all的意思

Linux中，以点.开头的目录/文件就是隐藏文件

默认我们直接ls 或ls -l 是无法显示隐藏文件的，但如果我们带上a也就是 ls -al 就可以显示隐藏文件了

我们自己也可以创建隐藏目录/文件，只要文件名以点.开头就行了。

例如 touch .readme.txt 和 mkdir .youcanseeme

ls / 查看根目录下的文件

ls /root 指定查看root目录下的文件

ls -ld /root 查root这个目录本身，而不是进入到这个目录，把这个目录里面的内容查出来

**ls -ld ** 查看目录本身

6. rmdir指令 && rm指令

rmdir 指定目录 删除指定的空目录

那我们应该如何删除不为空的目录呢？

Linux中的目录结构是多叉树结构，并且Linux中所有的目录结构是递归式定义的

所以我们删除的指令要带上r，r就是递归的意思 。rm -r

但是在root用户下，我们删除目录它会一直问我们是否要删除

如果我们希望强制删除，就还要带上f，f就是force强制的意思

rm -rf

如何删除指定的文件呢？

rm 文件名

我们发现它也要问我们是否删除，所以我们同样可以加上f选项来强制删除该文件

rm -f 文件名

rm -rf /

删除整个系统，可以在云服务器上重装

rm * -rf 删除当前目录下的所有

rm test{1..100}.o删除多个文件时

**方式二：**利用管道

表示按行统计

1

wc单词计数

在 Linux 中，wc 是 “word count”（单词计数）的缩写，是一个用于统计文本内容的命令行工具。它可以统计文件（或输入流）中的行数、单词数、字节数等信息

wc 命令的常用选项：

  • -l：仅统计行数（line count）
  • -w：仅统计单词数（word count）
  • -c：仅统计字节数（byte count）
  • 如果不指定选项，默认会同时输出 行数、单词数、字节数 和文件名（如果输入是文件）。

13. grep指令-行文本进行过滤

grep "main" ./*同时grep指令也支持通配符，表示在当前路径下找

常用的选项

  • -n显示行号

    grep -n "main" *.c将文件中包含main的，后缀为.c的显示，并带上行号

  • -i 忽略大小写

grep还可以查进程

3.沾滞位

1. Linux多用户下，对于文件的访问，是隔离的

2. 需求：我们想让Linux下不同的账号用户，共享文件呢？

共享文件不能在任何一个普通账号的家目录里面，只能在其它目录下，比如根目录下

但是根目录也是一个目录，所以新建时需要以超级用户身份来创建sudo，或者root用户来新建

默认下other是没有创建文件的权限的，所以需要sudo添加w

3. 问题：我们想既共享文件，同时other不能随便删除这个文件

我们想共享文件，也就是需要让在根目录下建的目录的other的w权限放开，

这样other就可以在这个共享目录中新建文件了，我们可以修改共享目录文件中新建的文件的属性也就是可以让拥有者和所属组的权限是可读可写可执行，other是不能读不能写不能执行

但存在一个问题，在目录中，创建删除修改一个文件是由目录的w权限决定的，与目标文件无关，也就是如果我们将共享目录的w权限放开了，那么other就可以删除我们共享目录里的文件了

添加粘滞位：sudo chmod +t /目录名/

任何用户，在设置了粘滞位的目录下

a. 自己只能删除自己的目录，其他人无法删除

b. root无视

c. 粘滞位，只能给目录设置，不需要给普通文件设置

Linux下默认给tmp目录加了粘滞位

#include <stdio.h>

int main()
{
  FILE* fp = fopen("/dev/pts/1", "w");
  if(fp == NULL)
  {
     perror("fopen");
     return 1;
  }

  char buffer[1024];
  while(1)
  {
     printf("Enter# ");
     scanf("%s", buffer);
     fputs(buffer, fp);
     fflush(fp);
  }
  
  fclose(fp);
  return 0;
}

指令 说明

reboot 现在重新启动计算机
sync 把内存的数据同步到磁盘
