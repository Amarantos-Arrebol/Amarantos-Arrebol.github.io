1.基本介绍

1.1进程间通信的理解

ls -l | wc -l

ls -l是一个进程，wc -l也是一个进程，ls -l进程通过管道将结果交给wc -l

为什么要进程间通信？

  • 数据传输：一个进程需要将它的数据发送给另一个进程
  • 资源共享：多个进程之间共享同样的资源。
  • 通知事件：一个进程需要向另一个或一组进程发送消息，通知它（它们）发生了某种事件（如进程终止时要通知父进程）。
  • 进程控制：有些进程希望完全控制另一个进程的执行（如Debug进程），此时控制进程希望能够拦截另一个进程的所有陷入和异常，并能够及时知道它的状态改变。

怎么通信？

进程间通信的本质：是先让不同的进程，先看到同一份资源[“内存”]（然后才有通信的条件）

同一份资源[“内存”] 得由操作系统提供—>系统调用—>操作系统的接口—>设计统一的通信接口

什么是通信？

进程间通信（IPC，Inter-ProcessCommunication）是指运行在同一台计算机或不同计算机上的多个进程之间进行数据交换和通信的技术。由于每个进程都有自己的地址空间，它们无法直接访问彼此的数据，因此需要通过特定的机制来实现通信。IPC是操作系统和多进程编程中的一个重要概念，广泛应用于分布式系统、多任务操作系统以及各种应用程序之间

进程间通信的发展

  • 管道
  • SystemV 进程间通信
  • POSIX 进程间通信

2.管道的基本介绍

当我们需要解决进程间通信这个问题时，首先想到的肯定是基于已有的技术来直接实现。最初的工程师们，也是这么思考的。

而文件，就是当时最合适的选择。因为文件被打开后，内核会为它创建一个 struct file 对象，这个对象天然自带缓冲区。我们以前写文件，是把缓冲区里的数据最终刷新到磁盘上；那反过来想，能不能不把数据刷到磁盘，而是让另一个进程直接读取这个缓冲区里的数据？这样一来，进程间通信的核心逻辑不就通了吗？

而且文件共享的原理我们也清楚：只要让两个进程持有同一个文件的文件描述符，本质上就是让它们的文件指针指向内核中同一个文件缓冲区 —— 这正好能支撑 “一个进程写、另一个进程读” 的通信场景。

匿名管道，就是基于这种文件机制实现的进程间通信方式。

  • 管道是 Unix 系统中最古老的进程间通信形式。
  • 我们可以通俗地理解：从一个进程连接到另一个进程的专属数据流，就叫做 “管道”。

2.匿名管道

2.1原理

2.1.1用 fork 来共享管道原理

进程间通信的本质，是先让不同的进程看到同一份资源（此处的资源即内存），具备这一条件后，通信才能够实现。匿名管道正是通过 fork 操作，让父子进程共享同一份内存资源（管道对应的内核缓冲区），从而实现通信。

匿名管道通常用于父子进程间的通信，其核心原理即依赖 fork 的浅拷贝特性：

父进程在创建子进程前，先打开管道（对应内核中的 struct file 对象与缓冲区）并持有其文件描述符；执行 fork 时，子进程会浅拷贝父进程的文件描述符表，从而继承该管道的文件描述符，与父进程共同指向同一个内核缓冲区。

需要明确的是，真正的管道与磁盘无关，无需将数据刷新到磁盘存储。

数据仅在管道对应的内存缓冲区中传递，这既是管道通信效率较高的原因，也是其与普通文件操作的核心区别

2.1.2站在文件描述符角度-深度理解管道站在内核角度-管道本质

管道是基于文件的，但管道是通信技术，所以还是要改代码

操作系统提供了一种内存级的文件，调用几个接口就能打开，支持读 / 写操作；父进程打开这个文件后 fork 创建子进程，子进程会拷贝父进程的文件描述符表，所以也能访问该文件；若要父子单向通信，关掉父进程的读权限、子进程的写权限，就能实现父写子读。

这个管道是被操作系统单独设计的（但是可以复用代码），因此要配上操作系统单独的系统调用**pipe**

#include <iostream>
#include <unistd.h>
#include <sys/types.h>
#include <unistd.h>
#include <sys/wait.h>

int main()
{
  // 1.创建管道
  int fds[2] = {0}; // fds[0]:读端  fds[1]:写端
  int n = pipe(fds);
  if (n < 0)
  {
    std::cerr << "pipe error" << std::endl;
    return 1;
  }
  std::cout << "fds[0]: " << fds[0] << std::endl;
  std::cout << "fds[1]: " << fds[1] << std::endl;

  // 2.创建子进程
  pid_t id = fork();
  if (id == 0)
  {
    // child
    // 3.关闭不需要的读写端，形成通信信道
    // 子进程->写，关闭读端
    close(fds[0]);

    close(fds[1]);
    exit(0);
  }
  // 3.关闭不需要的读写端，形成通信信道
  // 父进程->读，关闭写端
  close(fds[1]);

  waitpid(id, nullptr, 0);
  close(fds[0]);

  return 0;
}

至此，父子进程间并没有通信。我们前面做的这些工作，都只是进程间通信的前提条件，即让父子进程能看到同一份资源（仅仅是将父子间通信的信道建立起来了）

为什么要这么麻烦呢？因为进程具有独立性，彼此相互隔离，所以进程在通信之前，必须先建立信道，这个信道由操作系统提供

4.父子间通信

snprintf 是C语言提供的函数，能将指定内容格式化后写入目标缓冲区。

它的格式化规则和 printf 一致，往显示器上怎么输出，就可以用 snprintf 怎么格式化内容到缓冲区

形参：

  1. str：目标缓冲区。存放格式化后的字符串
  2. size：缓冲区最大字节数。含 \0 限制长度防越界 无需填满
  3. format：格式化规则。同 printf 含固定文本和占位符
  4. …：可变参数。对应占位符的实际数据

返回值：成功 返格式化字符串实际长度（不含 \0） 失败 返负数

write写文件：向特定文件描述符写入特定字符串和特定长度

返回值

  • 正数：实际成功写入的字节数（可能小于自己指定的 count，比如缓冲区暂时满了）
  • 0：几乎不会遇到（极少数情况是没数据可写，比如对方关闭了读端但未完全断开）
  • 负数：写操作出错（比如 fd 无效、管道另一端关闭、权限不够等）

read读文件：从特定的文件描述符里读数据到buf，读取count字节

返回值：

  • 正数：实际读到的字节数
  • 0：读到文件尾（比如管道另一端关闭，没数据可读了）
  • 负数：读操作出错（比如 fd 无效、缓冲区问题等）

#include <iostream>
#include <unistd.h>
#include <cstdio>
#include <cstring>
#include <sys/types.h>
#include <sys/wait.h>

void ChildWrite(int wfd)
{
  char buffer[1024];
  int cnt = 0;
  while (true)
  {
    // 格式化到buffer里，缓冲区最大字节数，格式化成什么
    snprintf(buffer, sizeof(buffer), "I am child, pid: %d, cnt: %d", getpid(), cnt++);
    // 将特定字符串写给父进程
    write(wfd, buffer, strlen(buffer)); // 用strlen，因为往文件里写的时候不写\0
    sleep(1);
  }
}

void FatherRead(int rfd)
{
  char buffer[1024];
  while (true)
  {
    buffer[0] = 0;
    // n是实际读到的字节数，sizeof(buffer) - 1是我们期望读到的字节数
    ssize_t n = read(rfd, buffer, sizeof(buffer) - 1); // 我们知道要读的是字符串，读的时候需要自己加\0
    if (n > 0)
    {
      buffer[n] = 0;
      std::cout << "child say: " << buffer << std::endl;
    }
  }
}

int main()
{
  // 1.创建管道
  int fds[2] = {0}; // fds[0]:读端  fds[1]:写端
  int n = pipe(fds);
  if (n < 0)
  {
    std::cerr << "pipe error" << std::endl;
    return 1;
  }
  std::cout << "fds[0]: " << fds[0] << std::endl;
  std::cout << "fds[1]: " << fds[1] << std::endl;

  // 2.创建子进程
  pid_t id = fork();
  if (id == 0)
  {
    // child
    // 3.关闭不需要的读写端，形成通信信道
    // 子进程->写，关闭读端
    close(fds[0]);
    ChildWrite(fds[1]);

    close(fds[1]);
    exit(0);
  }
  // 3.关闭不需要的读写端，形成通信信道
  // 父进程->读，关闭写端
  close(fds[1]);
  FatherRead(fds[0]);

  waitpid(id, nullptr, 0);
  close(fds[0]);

  return 0;
}

2.3管道的特性和通信情况

管道的5种特性

1）匿名管道，只能用来进行具有血缘关系的进程进行进程间通信（通常用于父子）

2）管道文件，自带同步机制

3）管道是面向字节流的

怎么读，和曾经怎么写，没有必然关系。

写的快，读的慢——>取决于自己想怎么读

4）管道是单向通信的（属于半双工的一种特殊情况）

  • 任何一个时刻，一个发，一个收 — 半双工
  • 任何一个时刻，可以同时发收 —全双工

5）（管道）文件的生命周期，是随进程的

管道的4种通信情况

1）写慢，读快 — 读端就要阻塞（进程），等写

2）写快，读慢 —写满了的时候，写就要阻塞等待（读端）

3）写关，继续读 —read就会读到返回值为0，表示文件结尾

4）读关闭，写继续 —写端再写入，没有意义，OS不会做没有意义的事，操作系统会杀掉写端进程，通过发送异常信号 13) SIGPIPE

#include <iostream>
#include <unistd.h>
#include <cstdio>
#include <cstring>
#include <sys/types.h>
#include <sys/wait.h>

void ChildWrite(int wfd)
{
  char c = 0;
  int cnt = 0;
  while (true)
  {
    write(wfd, &c, 1);
    printf("child: %d\n", cnt++);
  }
}

void FatherRead(int rfd)
{
  char buffer[1024];
  while (true)
  {
    sleep(100);
    buffer[0] = 0;
    // n是实际读到的字节数，sizeof(buffer) - 1是我们期望读到的字节数
    ssize_t n = read(rfd, buffer, sizeof(buffer) - 1); 
    if (n > 0)
    {
      buffer[n] = 0;
      std::cout << "child say: " << buffer << std::endl;
    }
    else if (n == 0)
    {
      std::cout << "n: " << n << std::endl;
      std::cout << "子进程退出了，我也退出" << std::endl;
      break;
    }
    else
    {
      break;
    }

    break;
  }
}

int main()
{
  // 1.创建管道
  int fds[2] = {0}; // fds[0]:读端  fds[1]:写端
  int n = pipe(fds);
  if (n < 0)
  {
    std::cerr << "pipe error" << std::endl;
    return 1;
  }
  std::cout << "fds[0]: " << fds[0] << std::endl;
  std::cout << "fds[1]: " << fds[1] << std::endl;

  // 2.创建子进程
  pid_t id = fork();
  if (id == 0)
  {
    // child
    // 3.关闭不需要的读写端，形成通信信道
    // 子进程->写，关闭读端
    close(fds[0]);
    ChildWrite(fds[1]);

    close(fds[1]);
    exit(0);
  }
  // 3.关闭不需要的读写端，形成通信信道
  // 父进程->读，关闭写端
  close(fds[1]);
  FatherRead(fds[0]);
  close(fds[0]);

  sleep(5);

  int status = 0;
  int ret = waitpid(id, &status, 0); // 获取到子进程的退出信息
  if (ret > 0)
  {
    printf("exit code: %d; exit signal: %d\n", (status >> 8) & 0xFF, (status) & 0x7F);
    sleep(5);
  }

  return 0;
}

管道的写入原子性

如果子进程向管道中单次写入的字节数小于 PIPE_BUF，那么子进程写入的过程就是原子的，即要么就写完，要么就不写。

    std::cout << "选择了一个子进程：" << c.Name() << std::endl;

    // 3. 将任务码发送给子进程
    c.Send(taskcode);
    std::cout << "发送了一个任务码：" << taskcode << std::endl;
  }

  // 让所有的子进程stop
  void Stop()
  {
    // 关闭父进程所有的wfd即可
    _cm.StopSubProcess();

    // 回收所有的子进程
    _cm.WaitSubProcess();
  }

  ~ProcessPool() {}

private:
  ChannelManager _cm;
  int _process_num; // 进程池里面进程的个数
  TaskManager _tm;  // 任务的管理
};

匿名管道只能用来进行具有血缘关系的进程进行进程间通信（通常用于父子）。

如果两个进程不相关？该如何进行通信呢？

不同的进程打开同一个路径下的同一个文件，内核中，操作系统不会把这个文件的inode/文件内容等，在内存中加载两次。因为不同的进程看到了同一个路径下的文件，而且这个文件是有名字的，因此叫命名管道。

路径具有唯一性

必须让Linux内核支持管道文件（只会被打开，不需要刷新）

如果我们想在不相关的进程之间交换数据，可以使用FIFO文件（即命名管道）来完成。

命名管道是一种特殊类型的文件。

3.1命名管道的使用

3.1.1指令的方式

mkfifo 创建管道文件

管道文件的文件类型以p开头

2）echo向管道文件中写入，输出重定向：echo "hello fifo">fifo

3）cat从管道文件中读，标准输入重定向，：cat < fifo

echo命令和cat命令运行的时候都是进程，所以我们进行的操作就是进程间通信

#include <string>
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#include "comm.hpp"

int main()
{
  umask(0);
  // 1.创建管道文件
  int n = mkfifo(FIFO_FILE, 0666); // 创建文件需要给文件带上权限
  if (n != 0)
  {
    std::cerr << "mkdir fifo error" << std::endl;
    return 1;
  }
  // 2.文件操作
  // 以只读的方式打开管道文件
  int fd = open(FIFO_FILE, O_RDONLY);
  if (fd < 0)
  {
    std::cerr << "open fifo error" << std::endl;
    return 1;
  }

  // 读管道文件
  char buffer[1024];
  while (true)
  {
    int n = read(fd, buffer, sizeof(buffer) - 1);
    if (n > 0)
    {
      buffer[n] = 0;
      std::cout << "client say# " << buffer << std::endl;
    }
  }

  close(fd);

  return 0;
}
