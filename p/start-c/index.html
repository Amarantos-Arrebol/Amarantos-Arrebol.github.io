#include <stdio.h>
#include <stdlib.h>

namespace dyy
{
	//命名空间中可以定义变量/函数/类型
	int rand = 10;

	int Add(int left, int right)
	{
		return left + right;
	}

	struct Node
	{
		struct Node* next;
		int val;
	};
}

int main()
{
	printf("%p\n", rand);

	// ::域作用限定符
	printf("%d\n", dyy::rand); // 访问命名空间里的变量

	dyy::Add(10, 20); // 访问命名空间里的函数

	struct dyy::Node node; // 访问命名空间里的结构体

	return 0;
}

1.2C++中的域

1）C++中的域有函数局部域，全局域，命名空间域，类域

2）C++规定，同一个域不能定义同名变量，不同域可以定义同名变量

3）默认先在局部域查找，局部域找不到就在全局域查找

4）命名空间域和类域可以解决命名冲突的问题

5）局部域，全局域会影响生命周期

1.3注意

1）默认先在局部域查找，找不到就在全局域查找，要找命名空间里的成员需要指定

2）局部域的只能在局部访问

3）项目工程中多文件中定义的同名namespace会认为是一个namespace，不会冲突，会自动合并

4）C++标准库都放在一个叫std(standard)的命名空间中

5）查找不到变量/定义，就编译报错：error C2065:“a”:未声明的标志符

02输入输出IO流

C++有一套自己的输入输出流

<iostream>是 lntput Output Stream的缩写，是输入输出流的头文件

<iostream>包含stdio.h，但在Linux下要包stdio.h

2.1基本介绍

使用c++输入输出不需要像printf，scanf 那样需要手动指定格式 （只有内存中才有整型，字符，浮点数等），如果需要控制小数点精度就使用printf

标准库中放在std的命名空间，c表示字符

  • std::cin标准输入流，是istream类的对象

    • 将字符串拿过来，再进行解析

  • std::cout标准输出流，是ostream类的对象

    • 访问cout的第一种方式就是要指定命名空间
    • 无论是什么类型，都把它转换成字符串再进行输出
    • （cout是console output的简写）

  • std::endl 是一个函数，end line的缩写

    • 流插入输出时，相当于插入一个换行字符加刷新缓冲区
    • 为什么我们更倾向于使用std::endl而不是C语言中的换行符 \n ？不同平台下换行符是不一样的

« 流插入，也可以叫输出，可以自动识别任意类型，让右边流向左边，支持连续的流插入，可以不同的类型

}

2.3 提高C++IO效率

1）cin cout的效率会低一些，printf ,scanf,cin,cout都是各自的一套输入输出，底层带缓冲区

std::cout « i « endl 先将右边的转成字符串，放到缓冲区，遇到一些刷新标志或主动刷新接口才会出去

典型的刷新标志：换行 主动刷新的接口（比如：flush）

2）C++的IO流互相有一些绑定关系，比如cin的时候外面有一个cout，cout外面遇到刷新标志，也会把它刷出去

在io需求比较高的地方，如部分大量输入的竞赛题中，加上以下3行代码，可以提高C++IO效率

ios_base::sync_with_stdio(false);//让C/C++不兼容，同步关系关掉

cin.tie(nullptr);//不再跟其他流进行绑定

cout.tie(nullptr);

5.2 引用的特性

1）引用在定义时必须初始化

2）一个变量可以有多个别名（引用），不能再变成其它变量的别名

3）引用一旦引用一个实体，就不能再引用其它的实体

4）引用不能替代指针，引用不能改变指向（在链表中，删除该结点，需要让上一个结点指向下一个结点），java的引用可以改变指向

5.3 引用的使用

可以给指针变量取别名

1）引用的第一个作用就是做参数减少拷贝，改变引用对象的同时改变被引用对象

函数传参时可以不传地址或者拷贝实参，而是将函数的形参写成实参的别名

例如：

2）引用的第二个作用 做返回值中减少拷贝，提高效率，减少临时拷贝

传值返回也会生成拷贝，会将要返回的值拷贝给生成的一个临时对象，再用这个临时对象做函数调用的返回值

使用指针返回，也可以修改，不过比较麻烦

**注意：**不是任何场景都能用引用返回

比如返回局部变量的引用，由于出函数，局部变量就销毁了，所以拿到引用（别名）也没用

越界不一定会报错

数组空间在堆上，没有free就不会销毁

vs中，数组越界是不一定会报错的，越界读不报错，越界写不一定报错，

一般是抽查，如果没报错，就是没有修改到抽查位

程序结束时，看是否被修改，修改了就一定越界

（读一般是获取值，然后打印；写是一种修改行为）

3）引用和指针不可相互替代，C++引用定义后不能改变指向，java的引用可以改变指向

5.7 指针和引用的关系

C++中指针和引用各有特点，互相不可替代，功能有重叠性

1）语法概念上，引用本质是给一个变量取别名，不开空间；指针是存储一个变量的地址，要开空间；

2）引用在定义的时候必须初始化，指针建议初始化

3）引用在初始化时引用一个对象后，就不能再引用其它对象；而指针可以不断地改变指向对象

4）引用可以直接访问指向的对象，指针需要解引用才能访问

5）sizeof的含义不同，引用结果为引用类型的大小，但指针始终是地址空间所站的字节个数

6）指针很容易出现空指针和野指针的问题，引用很少出现，引用使用起来相对安全一些

野引用的情况：引用做返回值

底层汇编的角度看，引用要开空间，引用也是用指针的实现的

6、inline

C++中的inline就是要替换C语言的宏

6.1宏函数

宏可以定义宏的常量，还可以定义宏函数

define 宏可以定义，宏本质是一种替换，ADD(a, b)会替换成((a) + (b))

#define ADD(a, b) a + b;

#define ADD(a, b) (a + b);

正确写法：#define ADD(a, b) ((a) + (b))不能加分号

int ret = ADD(1,2) ;//int ret = ((1) + (2));

不能加分号的原因：cout « ADD(1,2) « endl;

必须加外面的括号原因：cout « ADD(3, 5) * 2 « endl; //(3)+(5)*2

本来应该先算3+5，但是如果不加外面的括号就会先算5*2

必须加里面的括号原因：cout « ADD(x&y, x|y) « endl; //x&y+x|y

由于运算符优先级，所以会先计算y+x

缺点：复杂很容易出错，且不方便调试

**优点：**宏本质是一种替换，调用时不用建立栈帧（函数调用是要建立栈帧的）

6.2 inline 内联函数

C++中可以用inline修饰函数，放在函数返回值的前面，叫内联函数，

编译时C++编译器会在调用的地方展开内联函数，

这样调用内联函数就不需要建立栈帧，可以提高效率

  • inline对于编译器而言只是一个建议，也就是你加了inline，编译器也可以选择在调用的地方不展开

  • inline适用于频繁使用的短小函数，对于递归函数，代码相对多一些的函数，加上inline也会被编译器忽略

  • 如果函数内部包括循环，递归，代码量大且复杂，则不建议使用内联函数

1）

函数的调用本质是，call一个地址，这个地址是一个jmp指令，jmp指令再jmp一下就跳到函数的指令了

函数编译以后是一段指令，需要存储起来执行，第一句指令的地址就是函数的地址

内联函数没有地址

如果在汇编下函数走的是call，就没有建立栈帧，没有展开，内联没有起作用

2）

inline对于编译器而言只是一个建议，长的不展开，短的会展开（编译器的防御策略）

程序是一个个文件，通过编译会生成可执行程序

可执行程序会以一个进程的角度来进行运行，进程才会给它分配内存

内联函数缺点：代码膨胀 —>导致可执行程序变大，加载到进程也会变大

vs编译器在debug下面默认是不展开inline的，这样方便调试，短小函数在release下默认会展开

debug版本想展开需要设置两个地方

​ 1、项目右键->属性->C/C++->常规->调试信息格式改为程序数据库

​ 2、C/C++->优化->内联函数扩展改为只适用于inline

inline不建议声明和定义分离到两个文件，因为内联函数没有地址，分离会导致链接错误。

因为inline被展开，就没有函数地址，链接时就会出现报错

在类里的函数，默认是内联

应该将内联函数的声明和定义直接放到.h文件中

链接就是只有声明的函数去其它文件找他的地址
